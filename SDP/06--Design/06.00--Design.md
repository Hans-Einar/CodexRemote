# Design - Overview

**Status:** Draft v0.1  
**Date:** 2026-04-01  
**Source:** `SDP/05--DesignAnalysis/`

## 1) Purpose
This folder is the authoritative baseline design for CodexRemote.

These documents describe the intended system directly.  
They should not retain open analysis language once a choice has been made.

## 2) Design Documents

- `06.01--AppServerIntegration.md`
- `06.02--ThreadAndSessionModel.md`
- `06.03--BridgeAPI.md`
- `06.04--WorkspaceAndPreview.md`
- `06.05--WebUI.md`

## 3) Scope Rule
The design documents shall:

- define the chosen system structure
- state concrete integration and UI decisions
- provide a stable implementation reference

They shall not:

- rely on unstated protocol assumptions
- blur the boundary between bridge logic and UI logic
- present fallback behavior as if it were attached Mode B behavior
