# Design - Web UI

**Status:** Draft v0.1  
**Date:** 2026-04-01

## 1) Purpose
Define the first authoritative UI structure for CodexRemote.

## 2) Chosen Layout
The default desktop layout uses three functional regions:

- navigation for sessions and threads
- conversation view
- workspace explorer, editor, and preview

On narrow screens, these regions collapse into switchable panels.

## 3) Required UI Signals
The UI must display:

- current mode
- connection state
- selected workspace
- selected thread
- recent degraded-state or capability warnings
- terminal and editor status where those panels are active

## 4) Interaction Rules

- selecting a thread updates the conversation panel
- selecting a file updates the editor and preview panel
- sending a prompt always targets the currently selected thread
- fallback state must be visible before the user sends a prompt
- terminal execution must stay visually distinct from conversation messages

## 5) First-Scope Non-Goals

- advanced visual theming
- multi-workspace editing
- heavy editor features beyond preview

## 6) Current Component Direction
The current implementation direction prefers:

- a maintained tree-view component for workspace navigation
- a maintained terminal renderer for local shell execution
- a maintained Monaco-based editor for file editing
