# Implementation Plan - Web UI

## Goal
Implement the browser interface for thread selection, conversation view, workspace browsing, and preview.

## Planned Work

- create the initial responsive shell
- connect the shell to bridge APIs
- surface mode, connection, and degraded-state feedback clearly

## Exit Criteria

- the UI can browse threads and files
- the UI can send prompts once bridge support exists
- mode and connection state are visible to the user
