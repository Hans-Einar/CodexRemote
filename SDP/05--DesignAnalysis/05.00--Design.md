# Design Analysis - Overview

**Status:** Draft v0.1  
**Date:** 2026-04-01  
**Source:** `SDP/04--Architecture/04--Architecture.md`

## 1) Purpose
Capture the analysis, tradeoffs, and hardening work that leads into the authoritative CodexRemote design.

The documents in this folder may discuss:

- open protocol questions
- discarded options
- tradeoffs between Mode B and Mode A behavior
- decisions that still need validation

The final chosen design belongs in:

- `SDP/06--Design`

## 2) Design Documents

- `05.01--AppServerProtocol.md`
- `05.02--SessionAndThreadSync.md`
- `05.03--BridgeRuntime.md`
- `05.04--WorkspaceAndMarkdown.md`
- `05.05--WebUI.md`

## 3) Conventions

- reference requirement IDs from `SDP/03--Requirements/03--Requirements.md`
- harden open questions into explicit decisions before they move into `SDP/06--Design`
- record protocol risks explicitly when a decision depends on undocumented behavior
